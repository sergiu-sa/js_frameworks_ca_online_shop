# AI Usage Log

This document records how AI tools were used during the development of this project, in accordance with the course AI policy.

**Tool used:** Claude (Anthropic)

---

## Project Planning and Structure

**Date:** 1-2 March 2026
**Purpose:** At the start of the project, I used AI to brainstorm the overall scope and break the assignment down into manageable parts. Once I had a clear picture of the pages I needed to build, I asked for advice on how to organise the folder structure to follow Next.js App Router conventions and keep things clean as the project grew.
**Outcome:** Helped me land on a logical folder structure (pages, components, hooks, lib, types) before writing any code. All decisions were my own, AI just helped me think through the options.

## Code Quality and Best Practices

**Date:** Throughout the project (1-6 March 2026)
**Purpose:** I regularly used AI as a second pair of eyes for code review. This included checking for best practices in React/Next.js, making sure I was using hooks and state management properly, and getting suggestions for cleaner or more readable code.
**Outcome:** Improved overall code quality.

## Accessibility

**Date:** 5-6 March 2026
**Purpose:** Used AI to check that the UI followed accessibility best practices, things like proper ARIA labels, keyboard navigation, and screen reader support.
**Outcome:** Added aria attributes, proper semantic HTML, and accessible form error handling across the app.

## Debugging

**Date:** Throughout the project (1-6 March 2026)
**Purpose:** When I ran into bugs or unexpected behaviour, I described the issue to AI to help identify the root cause. For example: fixing form validation errors, resolving React hook warnings, and tracking down search filtering issues.
**Outcome:** Faster debugging. AI helped point me in the right direction.

## Writing and Naming

**Date:** Throughout the project (1-6 March 2026)
**Purpose:** To keep the codebase readable and consistent, I used AI for help with commit messages, code comments, and naming functions/variables. The goal was to use clear, conventional language that any developer could follow.
**Outcome:** More consistent and professional commit history and code documentation.

## General Guidance

**Date:** Throughout the project (1-6 March 2026)
**Purpose:** Whenever I felt stuck or unsure about a design choice, I used AI as a sounding board: to talk through options, weigh trade-offs, and make sure I was heading in the right direction.
**Outcome:** Better decision-making throughout the project. AI served as a mentor/advisor.
